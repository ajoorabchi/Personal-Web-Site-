Title: Configurable Recovery for Cooperative Design Transactions

Author:

Author:

Author:

Subject: Axel Meckenstock,Detlef Zimmer,Rainer Unland Configurable Recovery for Cooperative Design Transactions

Description: Transaction recovery is difficult in environments where work is performed in a cooperative style (e.g., design environments). We deal with this problem by defining cooperation and recovery primitives which are meant to be a solid foundation for application-specific protocols. Our approach relies on the transaction toolkit which gives us the flexibility to react to heterogeneous application requirements in nested transaction hierarchies. By allowing different nodes in a transaction hierarchy to run different protocols, we can control cooperation and thereby limit the extent of recovery. We illustrate our approach by examples from design environments. 1 Introduction 1.1 Overview Research work on recovery has emerged from techniques for simple transactions [HR83, BHG87] to techniques for nested transactions [Mos85, HR87, WS92].
