Title: Using Worldviews in Conceptual Object Oriented Systems Modelling

Author:

Subject: T. A. Wiggerts Using Worldviews in Conceptual Object Oriented Systems Modelling

Description: Research on object orientation flourishes in such a way that an object methodology jungle seems to be arising. Many methods clearly show that object orientation originates from programming languages: no formal definitions are given and concepts are blurred with implementational details. To get things clear there is a need for a formally defined framework that is on a conceptual level and supports different levels of abstraction. In this article we discuss how a number of features often said to be object oriented appear on a conceptual level. The notion of worldview is introduced to model inter object dependencies. This notion not only allows us to model pluggable architectures but also enables the derivation of polymorphism and subtyping. The importance of distinguishing levels of abstraction is stressed and wellformedness rules are presented to make sure worldviews respect the boundaries between these levels.
