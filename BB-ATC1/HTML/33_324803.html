Title: How to decide Query Containment under Constraints using a Description Logic

Author:

Author:

Author:

Subject: Ian Horrocks,Sergio Tessaris,Ulrike Sattler,Stephan Tobies How to decide Query Containment under Constraints using a Description Logic

Description: this paper we present a practical decision procedure for the case where neither the queries nor the constraints contain regular expressions. This represents a restriction with respect to the framework described in [2], where it was shown that the problem is still decidable if regular expressions are allowed in the schema and the (possibly) containing query, but this seems to be acceptable when modelling classical relational information systems, where regular expressions are seldom used [5, 4]. When excluding regular expressions, constraints imposed by EER schemas can still be captured, so the restriction (to contain no regular expressions) is only relevant to inter-schema constraints. Hence, the use of DLR in both schema and queries still allows for relatively expressive queries, and by staying within a strictly rst order setting we are able to use a decision procedure that has demonstrated good empirical tractability.
